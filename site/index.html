What it is

A save state for your AI's reasoning that works in any tool.

Continuum reads a conversation from any AI, keeps the verbatim text, the reasoning (decisions, rejected alternatives, open questions) and, on Cognee, a knowledge graph, then rebuilds that state anywhere. One neutral hub between every tool you use.

The category we created

Everyone stores facts. Only Continuum stores the reasoning.

Memory systems cover a thin slice: they recall knowledge. Continuum covers the whole surface, because it saves the working state of a decision and carries it to another AI.

How it works

Capture in flight. Rebuild anywhere.

One command captures the thinking; one command rebuilds it. Everything in between is Continuum's job.

CHECKPOINT

Save the thinking state

Continuum stores the conversation verbatim as the source of truth and extracts a reasoning graph: goals, decisions, rejected options, hypotheses, open questions, each timestamped.

continuum checkpoint -p billing < chat.txt
HOLD

Graph and verbatim, bounded

The reasoning graph is the index; verbatim is the fidelity. Salience based forgetting keeps it lean, so what comes back is compact, never a second context blowout.

continuum improve -p billing
RESUME

Rebuild in any AI

Get a paste ready resume package: the state, the rejected alternatives so it will not re propose them, and the exact words that matter, dropped into any provider.

continuum resume -p billing | pbcopy

Get started, zero to resumed in three minutes

Never used it before? Follow these six steps.

No account, no cloud, no keys. Everything below runs locally on your machine and works with any AI you already use.

INSTALL

Install Continuum

One command, Python 3.11 or newer.

pipx install continuum
SAVE

Save a conversation

Checkpoint a chat file, or inside an MCP connected AI just say save this whole chat to myproject with continuum.

continuum checkpoint -p myproject < conversation.txt
RESUME

Resume in any AI

Print the resume package and paste it into a fresh chat, same model or a different one.

continuum resume -p myproject
CHECK

Watch your context health

Mid conversation, see how safe your reasoning is and when to save.

continuum context -p myproject -m claude
MOVE

Switch platforms

Pull a chat straight from another tool, or export the whole state to paste into ChatGPT, Grok or Gemini.

continuum capture -p myproject --from grok --latest
DEPLOY

Go live for others

One blueprint puts the API and remote MCP server on Render, Fly or any Docker host, with a persistent disk and a token.

render blueprint, connect repo, deploy

Integrations

Add it to the AI you already use.

Continuum speaks MCP, so any MCP capable AI gets all sixteen verbs as tools. Or talk to it over plain HTTP.

Claude Code, Codex MCP stdio

Register the local server once, then say checkpoint this with continuum.

claude mcp add continuum -- continuum mcp

Claude web, Grok MCP remote

Run the streamable HTTP server and add its URL as a custom connector.

continuum mcp -t streamable-http \
  --host 0.0.0.0 --port 8771

Cursor, any MCP client MCP config

Point the client's MCP config at the command.

{ "mcpServers": {
  "continuum": { "command": "continuum",
                 "args": ["mcp"] } } }

HTTP API, your app REST

Run the server and call it from anything. Multi tenant via a user header.

continuum serve

GET  /export/{project}?format=md
